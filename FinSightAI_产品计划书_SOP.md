## Executive Summary（最后填写，但放最前）

**项目名称**：FinSight AI 智能投研终端（Web Application）

**做什么**：面向个人交易者的“对话式量化验证工具”。用户用自然语言提问（价差、相关性、简单策略回测等），系统通过 **LLM 语义映射 + MCP 原子化数据工具（TQSDK）+ Python Sandbox + 图表渲染** 产出可复现的分析结论、图表与可查看的代码（Glass Box）。

**为什么现在做（Why now）**：
- **技术窗口**：LLM 的多轮对话 + 代码生成/执行能力成熟，能把“想法→可运行分析”门槛显著降低。
- **用户行为变化**：个人交易者对“更快验证直觉/策略”的需求上升，但学习编程与数据工程成本高。
- **形态演化机会**：传统投研工具偏“专业化/手动操作”；我们切入“对话驱动 + 可复现 + 透明代码”新形态。

**为什么是我们做（Why us）**：
- **数据与执行链路能力**：通过 MCP 将 TQSDK 能力封装成原子化工具（`tq_get_kline` / `tq_get_quote`），降低 LLM 与前端集成复杂度。
- **产品抓手清晰**：以“灵感卡片墙 + 一键运行 + 可展开代码”的组合，兼容小白与进阶用户。

**规模预估（以假设为基础，需调研验证）**：
- **核心指标路径**：种子用户 → 激活（完成 1 次卡片运行）→ 留存（每周 ≥2 次分析）→ 付费（订阅/用量）。
- **收入上限估算模板**：年度收入上限 ≈ 可服务交易者数（SAM） × 付费率 × ARPU（详见“财务模型”与“市场规模估算”）。

**要花多少钱、多久回本（以 12 个月模型为例，需验证）**：
- 主要成本：研发人力（前后端/LLM/MCP/数据）、云算力（推理/执行/存储）、获客投放。
- 目标：第 6–9 个月达到单月收支平衡；第 12 个月形成稳定订阅收入与可控 CAC。

**主要风险**：
- 价值不被认可（“结论不可信/不透明”）→ 以 Glass Box、可复现、数据来源标注缓解。
- 合规与数据授权风险 → 明确数据来源/授权边界、审计与风控机制。
- 体验风险（错误率高/执行不稳定）→ 自我修复、可观测、沙箱隔离与配额。

**决策建议**：建议立项进入 4–6 周 MVP 预研与小规模内测，目标验证“高频场景 + 付费意愿 + 成本结构”三件事；通过后进入 3 个月产品化与商业化试点。

## 1. 概念与背景（Why this / Why now）

### 1.1 问题陈述（从 MCP_product.md 抽取）
- **目标用户**：个人期货/量化爱好者、半专业交易者（不一定会写代码）。
- **核心痛点**：
  - 想验证交易直觉（价差、联动、均线规律、简单回测），但不会写代码/不会找数据。
  - 传统工具要么学习成本高，要么黑箱，难以复现与自证。
- **核心理念**：你的想法，数据的答案（Your Idea, Data’s Answer）。

### 1.2 为什么是现在
- LLM 能完成“语义→代码→执行→总结”闭环。
- Web 形态能提供开箱即用体验，降低“插件/API”门槛，扩大可服务人群。

### 1.3 机会类型（Strategic role）
- **攻击型增量**：切入对话式投研新形态，争取订阅收入。
- **防御性**（可选）：在“LLM 原生投研”趋势下，避免被通用大模型/社区脚本替代。
- **实验性**：验证“透明化投研 + 轻回测”的付费意愿与成本结构。

### 1.4 影响（Impact）
- 对收入：新增订阅/用量付费收入；可扩展至企业版（研究/投顾/资管工具）。
- 对品牌：建立“可信、可复现、透明”的 AI 投研心智。

## 2. 市场与机会分析（Market & Opportunity）

> 说明：本章提供可直接落地的“估算框架 + 待补数据项”。如需补齐真实数据，可在调研阶段用公开报告/问卷/访谈填充。

### 2.1 目标细分市场（我们切哪一块）
- **细分**：国内期货为主的个人交易者与量化入门者。
- **场景切入**（MVP 高确定性）：
  - 价差/相关性快速验证（Spread & Correlation）
  - 跨期/跨品种套利分布（Arbitrage）
  - 极简策略回测（Simple Backtest：隔夜/均线/突破雏形）

### 2.2 市场规模估算（模板）
- **潜在用户数** = 国内个人交易者规模 × 其中愿意使用数字化工具比例 × 其中愿意尝试 AI 助手比例  
- **SAM（可服务市场）** = 潜在用户数 × 我们覆盖渠道可触达比例  
- **年度收入上限（理论）** = SAM × 付费率 × 年 ARPU  
- **需要验证的关键参数**：
  - 付费率（订阅 vs 用量）
  - ARPU（按档位）
  - CAC（获客成本）
  - 单用户月度推理/执行成本（COGS）

### 2.3 用户需求与心理动机（表层 vs 深层）
- **表层需求**：快速拿到数据与图；一键完成常见分析；少学/不写代码。
- **深层动机**：降低不确定性焦虑、提升“我能掌控/我懂”的安全感；获得效率与专业认同（能解释策略逻辑）。

### 2.4 趋势
- 投研工具从“手动操作”走向“对话驱动 + 自动执行”。
- 用户结构变化：更多轻量交易者/内容驱动型学习者，需要“可解释 + 可复现”。

## 4. 公司资源与战略匹配（Why us）

### 4.1 我们的关键优势
- **MCP 工具协议**：把 TQSDK 能力抽象成稳定原子工具，LLM 只需调用工具拿到 CSV/JSON。
- **执行沙箱**：Python Sandbox 让分析可复现、可审计、可管控。
- **前端呈现**：对标 Claude Artifacts/Jupyter 的“对话 + 画布”结构，适配投研工作流。

### 4.2 关键约束
- **合规/授权**：数据授权、用户交易建议边界（免责声明、风控提示）。
- **成本结构**：推理与沙箱执行成本可能随活跃增长快速上升，需要配额/缓存/分层计费。
- **可靠性**：代码执行失败率/重试与自我修复能力决定体验。

### 4.3 项目级 SWOT（模板化结论）
- **S**：数据工具原子化、透明可复现体验、垂直场景聚焦
- **W**：早期数据覆盖有限（10–20 品种）、品牌与渠道不确定
- **O**：对话式投研新形态窗口、个人交易者学习需求旺盛
- **T**：通用大模型替代、价格战、监管与舆情风险

## 7. 产品方案与演进路线（Product & Roadmap）

### 7.1 现有方案摘要（来自 MCP_product.md）
- 首页“灵感卡片墙”（一键运行）
- 对话与执行：LLM 语义映射 → MCP 获取数据 → Python 计算绘图 → 前端展示
- Glass Box：默认隐藏代码，支持展开查看分析逻辑
- MCP 核心工具：`tq_get_quote` / `tq_get_kline`（CSV/JSON 返回）

### 7.2 分期路线图（建议三期）

#### 一期（4–6 周）：MVP + 核心闭环
- **必须上线**：
  - 灵感卡片（3–6 个高频模板：相关性/价差、跨期分布、隔夜收益等）
  - 对话输入 + 画布呈现（静态图 + 结论）
  - MCP：`tq_get_kline` / `tq_get_quote`
  - Python Sandbox：基础绘图（Matplotlib）、结果缓存、失败自我修复（最小可用）
  - Glass Box：查看代码 + 数据表 Top5 预览 + CSV 下载
- **关键假设**：
  - 用户愿意为“省时间 + 可信复现”付费
  - 卡片能显著提高激活与留存
- **对应指标**：激活率、运行成功率、首周留存、试付费意向

#### 二期（6–12 周）：扩场景 + 提升可用性
- **扩展**：品种覆盖从 10–20 → 全量国内期货（或优先 TOP 品种）
- **交互**：ECharts 交互图（缩放/拖拽/指标叠加）、模板参数化（时间窗/周期/对比品种）
- **可信与效率**：更强的错误提示、成本控制（缓存/配额/队列）、可观测（链路追踪）
- **验证**：付费转化、价格敏感度、单位经济（单用户毛利）

#### 三期（3–6 个月）：平台化/生态化
- **开放**：用户自定义“分析模板”（可分享/可市场）
- **团队版**：协作、项目空间、权限、审计、导出报告
- **企业能力**：SSO、私有化/专有云、合规模块

### 7.3 MVP 功能清单（按“用户-场景-价值-指标”）
- **灵感卡片**
  - 用户：小白/入门
  - 场景：不知道问什么
  - 价值：一键得到可复现结果
  - 指标：卡片点击率、卡片运行完成率、激活率提升
- **合约语义映射（System Prompt 知识库）**
  - 用户：所有
  - 场景：自然语言提品种
  - 价值：减少错误 symbol、提升成功率
  - 指标：映射命中率、错误率
- **数据获取（MCP 工具）**
  - 用户：所有
  - 场景：取 K 线/元数据
  - 价值：稳定、可控、可审计的数据入口
  - 指标：调用成功率、时延、缓存命中
- **Glass Box（查看代码/数据）**
  - 用户：进阶/谨慎型
  - 场景：验证结论可信
  - 价值：建立信任与学习
  - 指标：展开率、投诉率下降、留存提升

## 8. 商业模式与定价策略（Business Model & Pricing）

### 8.1 收入来源（短中期）
- **C 端订阅**：按月/年（基础版/专业版）
- **用量付费（补充）**：按“运行次数/沙箱分钟数/高级数据调用”计费（防止重度用户把成本打穿）
- **B 端团队版（中期）**：SaaS 订阅 + 增值服务（集成/私有化）

### 8.2 定价结构（建议草案，需调研验证）
- **Free**：每日/每周有限运行次数；仅基础模板；低并发；基础图表
- **Pro（个人）**：无限或较高配额运行；全模板；更快队列；高级图表；更长历史数据；导出报告
- **Team/Enterprise**：共享配额、协作空间、审计、SSO、私有部署选项

### 8.3 定价逻辑（怎么讲清）
- 价值锚点：省掉找数据/写代码/调参时间 + 结果可复现可信
- 对标：传统投研订阅、量化平台订阅、通用大模型订阅
- 行为引导：Free 让用户完成“第一张图”；Pro 覆盖高频验证；Team 覆盖协作与合规

### A1. 机会分析
- **目标**：把直觉想法收敛成可讨论的产品机会。
- **输入**：用户痛点、管理层要求、趋势观察。
- **输出**：《机会分析一页纸》+ 问题陈述草稿。
- **操作要点**：Timing / Strategic role / Impact 三问。
- **自检清单**：能否 3–5 句话讲清“谁的什么问题”；不做的损失；角色是否明确。

### A2. 市场与竞品分析
- **目标**：证明机会真实、规模足够、有成熟参考。
- **输入**：机会描述。
- **输出**：市场章节草稿 + 竞品对比表。
- **操作要点**：规模与成长、细分结构、动机、趋势；竞品卖点/渠道/定价/优劣。
- **自检清单**：是否写清估算公式；至少 1 个标杆拆解；明确切入细分。

### A3. 资源与战略匹配
- **目标**：说明为什么我们更有胜算。
- **输入**：市场与竞品结论。
- **输出**：Why us 章节 + SWOT。
- **操作要点**：用户资产/渠道支付/品牌文化/内部能力与约束。
- **自检清单**：能否一句话回答；约束是否明确。

### A4. 愿景/定位/目标-KPI
- **目标**：形成清晰产品坐标与 1 年指标。
- **输入**：机会+市场+资源。
- **输出**：愿景 2–3 句、定位句、KPI 表。
- **操作要点**：区分业务/用户/过程指标；带时间刻度。
- **自检清单**：是否具体；是否可量化。

### A5. Roadmap
- **目标**：先后顺序与阶段验证假设。
- **输入**：愿景/定位/KPI。
- **输出**：分期路线图 + MVP 清单。
- **操作要点**：每期：时间/必须功能/验证假设/指标。
- **自检清单**：一期是否闭环；取消一期后二三期是否站不住。

### A6. 价值与购买意愿验证（含定价）
- **目标**：验证价值认同与价格敏感度。
- **输入**：前述结论 + 原型/样品 + 价格假设。
- **输出**：不同价格点意愿数据 + 定性反馈 + 调整建议。
- **操作要点**：具体场景、取舍题、尽量接近真实付费。
- **自检清单**：至少两个价位数据；是否反哺定价与指标。

### A7. 商业模式与定价
- **目标**：从谁收钱、怎么收、收多少。
- **输入**：路线图 + 竞品定价 + 调研结论。
- **输出**：收入来源清单 + 定价结构。
- **操作要点**：计价单位、梯度、组合空间、定价逻辑。
- **自检清单**：能解释“为什么这个价”；是否考虑组合。

### A8. 运营与增长
- **目标**：获客、验证、持续增长。
- **输入**：路线图&商业模式&渠道资源。
- **输出**：阶段策略 + 增长实验清单。
- **操作要点**：冷启动路径；3–6 个月增长指标；2–3 个重点策略。
- **自检清单**：第一批用户来源；至少 2 个可 A/B 假设。

### A9. 组织与项目计划
- **目标**：Owner 清晰、人手够、节奏可执行。
- **输入**：前述输出。
- **输出**：组织分工 + 人力规划 + 里程碑。
- **操作要点**：自研/外包划分；扩编点；需求-开发-测试-上线节奏。
- **自检清单**：是否有无人负责；节奏是否现实。

### A10. 财务模型
- **目标**：值不值得投、多久回本。
- **输入**：定价&路线图&人力。
- **输出**：12–24 月模型 + 峰值投入/回本点。
- **操作要点**：最小模型；显式假设；敏感变量。
- **自检清单**：关键参数下滑 30% 是否可接受；假设是否清晰。

### A11. 风险分析
- **目标**：识别做砸点并给对策。
- **输入**：全部内容。
- **输出**：风险矩阵。
- **操作要点**：覆盖市场/产品/商业/执行/合规运营。
- **自检清单**：高风险是否都有具体对策。
